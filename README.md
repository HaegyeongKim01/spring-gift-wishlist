# spring-gift-wishlist

## 🚀 유효성 검사 및 예외 처리
### 기능 요구 사항

🗣️상품을 추가하거나 수정할 때, 클라이언트로부터 잘못된 값이 전달될 수 있다.

잘못된 값이 전달되면 클라이언트가 어떤 부분이 왜 잘못되었는지 인지할 수 있도록 응답을 제공한다.

- [X] 상품 이름: 공백 포함 최대 15자, Null 허용X
- [X] 특수 문자
    - 가능: (), [], +, -, &, /, -
    - 그 외 특수 문자 불가
- [X] “카카오”가 포함된 문구는 담당 MD와 협의한 경우에만 사용 가능
- [X] 가격 입력 Null 허용X
